## 2. 이기는 프로젝트의 조건
 
### ✅ 필수 요소

- 이미 알려진 공격만 재현 (새로운 각도 없음)
- 데모 없는 아이디어 발표

## 3. 추천 프로젝트 방향
 
### 🥇 최우선 추천: 자동화된 레드팀 CLI
 
```
입력: 타겟 LLM API endpoint (URL + key)
출력: 취약점 리포트 (탈옥 성공률, 인젝션 가능 경로, 심각도 등급)
```
 
**구조 예시:**
```
[공격 프롬프트 생성] → Claude API
        ↓
[타겟 모델 공격]    → Gemini API (제공된 4개 키 활용)
        ↓
[결과 분석 & 분류]  → Claude API
        ↓
[리포트 생성]       → CLI 출력 / HTML 리포트
```
 
**왜 강력한가:**
- Gemini API 키가 4개 제공됨 → 타겟 모델로 바로 활용 가능
- Developer Tooling 트랙과도 겹쳐서 어필 가능 (CLI 형태)
- AIM Intelligence의 Stinger 제품과 보완적 포지셔닝 가능
---
 
### 🥈 차선 추천: 가드레일 우회 탐지 파이프라인
 
기존 가드레일 시스템이 **탐지 못 하는 공격**을 자동으로 생성하고 검증:
 
- Adversarial prefix injection
- Many-shot jailbreaking
- 언어 전환 기반 우회 (한국어 ↔ 영어 혼용)
- Base64 / 코드 인코딩 우회
---
 
### 🥉 차별화 옵션: 멀티모달 탈옥 탐지
 
이미지 + 텍스트 조합 공격 탐지 — 희귀하고 임팩트 큼  
(Dasol Choi의 VLM 연구 배경과 직접 연결)
